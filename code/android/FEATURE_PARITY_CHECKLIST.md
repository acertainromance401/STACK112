최종 업데이트: 2026-06-03
기준 문서(iOS): `code/ios/README.md`, 루트 `README.md`

## 1) 릴리스 차단 기준

- [ ] Android 앱이 완전 온디바이스 모드(핵심 흐름에서 백엔드 필수 의존 없음)로 동작
- [ ] 실기기 Android에서 전체 사용자 흐름(E2E) 동작 확인
- [ ] Play Console 내부 테스트 트랙 배포 완료
- [ ] 개인정보처리방침 URL이 게시되어 있고 실제 앱 동작과 일치
- [ ] Data Safety 양식이 실제 수집/공유 동작과 일치

## 2) 핵심 기능 동등성 (iOS -> Android)

### 2.1 메인 내비게이션 (탭 5개)
- [ ] 홈 탭
- [ ] OCR 탭
- [ ] 검색 탭
- [ ] 복습 탭
- [ ] 마이페이지 탭
- [ ] 탭 전환 시 하단 내비게이션 상태 보존

### 2.2 OCR 및 스캔 라이브러리
- [ ] 카메라/사진 가져오기 기반 스캔 진입 흐름 구현
- [ ] 한국어 OCR 텍스트 추출 동작
- [ ] 스캔 이력 목록(라이브러리) 구현
- [ ] 선택한 스캔 상세 진입 시 크래시 없음

### 2.3 법률 분석 및 검색
- [ ] 핵심 법률 도메인 분류 동작
- [ ] IR 추출 파이프라인이 안정적인 구조화 출력 생성
- [ ] 로컬 판례 검색 결과 정확성 확보
- [ ] 로컬 인덱스 기반 유사 판례 추천 동작

### 2.4 AI 요약 및 해설
- [ ] AI 요약 화면에 로딩/성공/폴백 상태 제공
- [ ] 생성 결과 정제(placeholder/프롬프트 누출 없음)
- [ ] 시험 포인트 중심 요약 표시
- [ ] 모델 미사용 가능 시 규칙 기반 폴백 응답 제공

### 2.5 OX 퀴즈
- [ ] 현재 문맥 기반 OX 퀴즈 생성
- [ ] 동적 토큰/길이 제어의 동등 동작
- [ ] 정오답 피드백이 명확하고 즉시 제공
- [ ] 오답 저장이 안정적으로 수행

### 2.6 복습 및 개인화
- [ ] 오답 노트 목록 제공(Weak OX 리스트 동등)
- [ ] 자주 틀리는 영역 카드/목록 제공
- [ ] 약점 키워드 주입 또는 동등한 개인화 동작
- [ ] 복습에서 퀴즈/검색으로 재학습 흐름 연결

## 3) 모델 및 온디바이스 런타임 동등성

- [ ] Android 로컬 LLM 런타임 통합(llama.cpp Android/JNI 또는 동등)
- [ ] GGUF 모델 로딩 정책 정의(번들 포함 또는 최초 실행 다운로드)
- [ ] 모델 미존재 상태에서 사용자 친화적이며 비차단 UX 제공
- [ ] 요약/OX/검색 사전용 응답 캐시 존재
- [ ] 최소 1개 중급 Android 실기기에서 메모리/성능 검증 완료

## 4) 데이터 및 저장소 동등성

- [ ] iOS 데이터 모델을 Android Room/DataStore 스키마로 매핑
- [ ] 기존 법률 사전/판례 에셋 로컬 로딩 동작
- [ ] 스키마/버전 변경 시 마이그레이션 전략 문서화
- [ ] 앱 재시작 후 스캔/이력/오답노트 데이터 영속성 검증

## 5) UX 및 품질 게이트

- [ ] 목표 디바이스에서 콜드 스타트 시간 허용 범위 충족
- [ ] 30분 탐색 테스트에서 치명적 크래시 없음
- [ ] 모든 핵심 화면의 빈 상태/오류 상태/오프라인 상태 구현
- [ ] 접근성 기본 요건(글자 확대, 대비, 터치 영역) 충족
- [ ] 한국어 텍스트 렌더링 및 줄바꿈 품질 검증
