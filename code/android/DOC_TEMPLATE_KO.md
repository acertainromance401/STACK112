# 문서 제목

최종 업데이트: YYYY-MM-DD
작성자: 이름
대상 버전: Android v0.1.0

## 1. 문서 목적

- 이 문서가 해결하려는 문제를 한 문장으로 작성합니다.
- 범위(포함/제외)를 명확히 적습니다.

## 2. 배경

- 관련 기능 또는 이슈 링크를 적습니다.
- iOS 동등성 기준이 있다면 기준 문서를 연결합니다.

## 3. 요구사항

- 기능 요구사항을 체크리스트로 작성합니다.
- 비기능 요구사항(성능, 안정성, 접근성)을 분리합니다.

예시:
- [ ] 핵심 사용자 흐름이 중단 없이 동작
- [ ] 오프라인 상태에서 필수 기능 제공
- [ ] 치명적 크래시 0건

## 4. 구현 설계

### 4.1 아키텍처
- 모듈 구조
- 데이터 흐름
- 상태 관리 방식

### 4.2 주요 컴포넌트
- 화면/뷰모델
- 데이터 계층
- 로컬 모델/런타임

## 5. 테스트 계획

- 단위 테스트
- 통합 테스트
- 실기기 시나리오

예시 시나리오:
1. OCR 진입 -> 분석 -> 요약 -> OX -> 복습
2. 백그라운드 전환 후 복귀
3. 저메모리 상황 폴백 확인

## 6. 배포 체크

- Play Console 입력값 점검
- 버전 정책(versionCode/versionName) 점검
- 내부 테스트 트랙 배포 여부

## 7. 리스크 및 대응

- 예상 리스크를 우선순위 순으로 정리
- 리스크별 대응 방안을 1:1로 작성

## 8. 변경 이력

- YYYY-MM-DD: 초안 작성
- YYYY-MM-DD: 리뷰 반영
