# AI SYS — UX/UI 대대적 개편 명세서

> 발행일: 2026-05-11  ·  대상: iOS (SwiftUI · SwiftData)  ·  디자인 토큰 파일: `code/ios/AISYSApp/Sources/DesignSystem.swift`
> 본 문서는 "경찰 고시 수험생을 위한 AI 기반 학습 보조 앱" 의 1차 UX 개편 결과를 정리한다. 화려한 마케팅용 디자인 시스템이 아니라, **합격할 때까지 버티게 만드는 도구**로서의 기능 우선 명세이다.

---

## 0. 디자인 철학 (One Liner)

> **화려함보다 집중 / 탐색보다 반복 / 끝까지 버티게 만드는 도구.**

| 원칙 | 적용 방식 |
|---|---|
| 정보 밀도 ↑ 시선 피로 ↓ | 다크 네이비 배경 + 골드 1색 포인트. 그라데이션·그림자 최소화. |
| 클릭 3회 이내 학습 시작 | 홈 → "오늘의 문제 시작" 1탭 → 풀이 화면. |
| 회독 사이클 닫힘 | 풀이 → 오답 자동 저장 → AI 분석 → 추천 회독 → 다시 풀이. |
| 자기보고 기반 분석 | 확신도(확실/애매/찍음) 입력 → 찍어 맞춘 정답도 "약점" 으로 분류. |
| Gamification은 절제 | streak, D-Day, 누적 문항 수만 허용. 캐릭터·레벨·보상 금지. |

## 1. UX 구조 (Information Architecture)

### 1.1 6탭 메인 네비게이션

```
┌──────────────────────────────────────────────────────────────────┐
│  홈        문제풀이    오답노트    판례카드    AI분석    통계      │
│  house    pencil      bubble      stack      brain     bar.chart │
└──────────────────────────────────────────────────────────────────┘
```

| Tag | 화면 | 핵심 책임 |
|----|----|----|
| 0 | 홈 (`HomeView`) | D-Day, 오늘의 학습, 추천 복습, 최근 약점, AI 루틴 |
| 1 | 문제풀이 (`PracticeView`) | OX 풀이 + 확신도 입력 + 즉시 채점/해설 |
| 2 | 오답노트 (`WrongNoteView`) | 자주 틀리는 영역, 최근 오답 리스트, 자동 분류 |
| 3 | 판례카드 (`CaseCardsView`) | 스와이프 카드, OCR 진입(FAB) |
| 4 | AI분석 (`AIAnalysisView`) | 한 줄 진단, 약점 패턴, 확신도 분석, 회독 추천 |
| 5 | 통계 (`StatsView`) | 누적 메트릭, 7일 정답률 차트, 설정(D-Day/목표/백엔드) |

### 1.2 부 화면 / 모달

- **OCR**: 판례카드 탭의 FAB(`+`) → `fullScreenCover` 진입 → 추출 결과 → `ScannedCase` 저장 → 즉시 카드 덱에 반영.
- **CaseSummaryView**: 판례카드 탭에서 카드 탭/리스트 항목 탭 → `NavigationStack` push.
- **WeakOXListView**: 오답노트의 약점 영역 셀 → push.
- **D-Day / 목표 설정 시트**: 통계 탭 설정 카드의 행을 탭 → `.sheet`.

### 1.3 학습 회독 사이클 (Closed Loop)

```
┌─────────┐    ┌────────────┐    ┌──────────────┐    ┌──────────┐
│  홈    │ →  │ 문제풀이   │ →  │ 오답 자동저장 │ →  │ AI분석   │
│ 위젯   │    │ +확신도    │    │ + 분류       │    │ 약점 추천 │
└────┬───┘    └────────────┘    └──────┬───────┘    └────┬─────┘
     │                                  │                 │
     │           ┌──────────────────────┘                 │
     │           ▼                                        │
     │   ┌──────────────┐                                 │
     └──→│ 판례카드 회독 │ ←──── 추천 회독 카드 ────────────┘
         └──────────────┘
```

## 4. 간격 / 모서리 / 컴포넌트

| 토큰 | 값 |
|---|---|
| `AppSpace.xs / s / m / l / xl / xxl` | 4 / 8 / 12 / 16 / 24 / 32 |
| `AppRadius.s / m / l / pill` | 8 / 12 / 16 / 999 |

### 4.1 핵심 컴포넌트

- **`AppCard`** — 모든 정보 컨테이너의 기본형. `surface` 배경 + 0.5pt `separator` 외곽선.
- **`MetricBlock`** — 큰 숫자 + 라벨. 홈 / 통계의 4-칸 그리드 단위.
- **`AppTag`** — 11pt 칩. 단원·과목·메타.
- **`SectionHeader`** — 좌측 타이틀 + 우측 보조 액션 라벨.
- **`QuickActionButton`** — 홈 화면의 아이콘 3분할 바로가기.
- **`ConfidenceChip` (3종)** — 확실(success) / 애매(accent) / 찍음(danger). 선택 시 1.2pt 외곽선 + 25% 채움.

자동 분류 휴리스틱 (다음 버전 v2 예정):
- 문제 본문에 "다만/단/그러나" 가 포함 → **함정선택지**
- 확신도 = `sure` 인데 오답 → **개념혼동**
- 확신도 = `guess` 인데 오답 → **암기실패**
- 풀이 시간 데이터가 추가되면 → **시간부족**

## 6. 사용자 플로우 (시나리오)

### 시나리오 A: 출근 지하철 15분

1. 홈 진입 → D-Day 확인 (1초)
2. "오늘의 문제 시작" 탭 → 문제풀이 진입 (1탭)
3. 4문항 풀이, 매 문항 확신도 입력 (3분/문항)
4. 오답 1건 자동 저장
5. 세션 종료 → 홈 진입 시 진행률 4/30 로 갱신

### 시나리오 B: 약점 정리 30분

1. 홈 → "최근 약점" 카드 탭 → AI 분석 화면
2. 한 줄 진단 확인 → 가장 큰 약점 영역 식별
3. 오답노트 탭 → 같은 영역의 자주 틀리는 행 탭 → `WeakOXListView`
4. 해당 영역의 과거 오답 카드 회독

### 시나리오 C: 새 판례 학습

1. 판례카드 탭 → FAB(+) → OCR → 텍스트 인식
2. 자동으로 `LocalIRPipeline` 이 키워드/문장 추출 → `ScannedCase` 저장
3. 카드 덱 상단에 자동 등장 → 좌우 스와이프로 회독
4. 문제풀이 탭으로 이동 → 자동으로 최신 스캔본 기반 OX 4문항 생성

## 7. 집중력 유지 설계

| 설계 항목 | 적용 |
|---|---|
| 색 노이즈 최소화 | 다크 네이비 70%, 골드 ≤5% — 시선 피로 감소 |
| 애니메이션 절제 | 시스템 기본 외 커스텀 모션 없음 |
| 인지 부하 ↓ | 1화면 1 CTA — 홈은 "오늘의 문제 시작" 하나만 가장 크게 |
| 즉각 피드백 | 답안 제출 → 0.3초 내 정답/해설 카드. 모달 없음 |
| 회피 차단 | 오답이 즉시 저장되므로 "다음에 정리" 없음 |
| 진행 가시성 | 진행률 바 + 메트릭 3종으로 학습량 체감 |

## 8. 반복 학습 최적화

- **자동 약점 누적**: 영역별 오답 빈도 ≥2 이면 `weakSubjects()` 에 등재.
- **확신도 기반 약점 확장**: `guess` 로 정답을 맞춰도 분석 화면이 별도 카운트.
- **스캔본 기반 OX 자동 생성**: 마지막 스캔본만으로 즉시 4문항 출제.
- **회독 사이클 자동 닫힘**: 오답 → 오답노트 → 약점 → AI 추천 → 같은 영역 OX 재출제 (v2).

## 10. 다크모드 최적화 가이드

본 앱은 **다크 우선** 으로 설계되며, 라이트모드는 SwiftUI 의 자동 색 변환에 의존한다.

- 배경 명도 단계 3단(`background` < `surface` < `surfaceElevated`) — 카드 위계 시인성 확보.
- 본문 대비: `textPrimary(#F5F8FB)` vs `background(#0A1428)` 컨트라스트 ≈ 16:1.
- 보조 대비: `textSecondary(#9CA7BA)` vs `background` ≈ 7:1.
- 골드 가독성: `accent(#F5C418)` vs `background` ≈ 11:1 — 강조에 충분.
- 의미적 컬러는 모두 채도 60% 이하로 낮춰 시선 피로 완화 + 색맹 사용자 대응 (의미는 항상 아이콘과 병행).

---

## 11. SwiftUI 방향성

- `NavigationStack` 만 사용, `NavigationView` 폐기.
- 모든 화면은 `.withAppBackground()` modifier 로 일관 적용 → 글로벌 다크 톤 보장.
- 색·간격·폰트는 `AppColor / AppSpace / AppFont` 로만 호출 — 하드코딩 금지.
- 상태 저장: `SwiftData` (`ScannedCase`) + `UserDefaults` (`ReviewStore` / `StudyStore`). 외부 백엔드 의존 없음.
- LLM 호출은 `LLMService.shared` 만, OCR 후처리는 `LocalIRPipeline` 만.
