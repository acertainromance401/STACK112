# ADR 0001: On-device First 아키텍처 채택

- Status: Accepted
- Date: 2026-05-14
- Deciders: @acertainromance401

## Context
- 서버 비용 및 운영 복잡도를 낮추면서도 판례 검색/요약/OX 기능을 안정적으로 제공해야 했다.
- 심사 및 배포 사이클에서 서버 의존도를 줄일 필요가 있었다.

## Decision
- iOS 앱의 핵심 학습 플로우를 온디바이스 우선으로 동작하게 구성한다.
- 백엔드가 없어도 검색, IR 추출, 유사 판례, 요약, OX 생성이 가능해야 한다.

## Alternatives Considered
- 클라우드 백엔드 중심 RAG 구조
- 하이브리드(기본 서버 + 일부 오프라인)

## Consequences
- 장점: 네트워크/서버 장애 영향 최소화, 운영비 절감, 오프라인 사용성 확보
- 단점: 앱 번들 크기 증가, 단말 성능 의존, 모델 관리 복잡도 증가

## Follow-up
- 저사양 단말 대응 성능 튜닝 지속
- 모델/프롬프트 품질 회귀 테스트 루틴 유지
