# ADR (Architecture Decision Record)

ADR은 기술 선택의 배경과 트레이드오프를 기록하는 문서입니다.

## ADR 목록

- `0001-ondevice-first.md`
- `0002-governance-docs-baseline.md`

## 작성 규칙

- 파일명: `NNNN-title.md`
- 예시: `0001-ondevice-first.md`
- 상태: Proposed, Accepted, Superseded, Deprecated

## 템플릿

```md
# ADR NNNN: 제목

- Status: Proposed
- Date: YYYY-MM-DD
- Deciders: @owner

## Context
- 어떤 문제를 해결하려는가?

## Decision
- 무엇을 선택했는가?

## Alternatives Considered
- 대안 A
- 대안 B

## Consequences
- 장점
- 단점/리스크

## Follow-up
- 후속 작업
```

실제 템플릿 파일: `docs/adr/template.md`
