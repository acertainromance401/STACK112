# 사용자 평가 보고서

작성일: 2026-06-19
작성자: AI_SYS 팀
버전: v1.0

## 1. 평가 개요

본 평가는 앱의 전반적인 사용 경험, 디자인 완성도, 기능 이해도, 판례 요약 기능의 실사용성을 중심으로 의견을 수집하였다. 평가 결과, STACK112는 학습용 앱으로서의 디자인 컨셉이 명확하고 주요 기능의 흐름이 비교적 직관적이라는 긍정적인 반응을 얻었다.

## 2. 평가 대상

- 대상 사용자: 경찰·법학 시험 준비생 및 관련 학습자, 그리고 서비스와 무관한 일반 사용자 포함
- 평가 환경: iOS 앱 프로토타입을 실제 사용 환경과 유사하게 구성
- 주요 기준: 학습 흐름, 기능 이해도, 요약 결과의 실용성

## 3. 평가 방법

- 시나리오 기반 사용 과제
  - 판례 검색, 요약 확인, 문제 풀이, 오답 복습 흐름을 실제로 사용
- 정성 피드백 수집
  - 사용 중 혼란 요소, 기대 대비 만족도, 개선 요청 내용 인터뷰
- 정량 평가
  - 기본 만족도 점수, 사용 직관성, 기능 이해도

## 4. 사용자 경험 평가

### 4.1 전반적 인식

사용자 평가는 앱의 시각적 완성도, 기능 흐름, 판례 요약의 실용성 중심으로 진행되었다. 결과적으로 STACK112는 학습용 앱으로서 디자인 컨셉이 명확하고 기능 흐름이 비교적 직관적이라는 평가를 받았다.

### 4.2 화면 구성 및 정보 배치

화면 구성과 색감, 정보 배치가 일관되어 앱의 정체성이 잘 드러난다는 평가가 있었다.

### 4.3 주요 기능 이해도

처음 사용하는 사용자도 주요 기능을 비교적 쉽게 파악할 수 있어 진입 장벽이 낮다는 의견이 있었다.

### 4.4 판례 요약의 실사용성

판례 요약 기능은 단순히 내용을 줄이는 수준을 넘어, 학습자가 이해하기 쉽게 핵심 내용을 정리해 준다는 점에서 긍정적인 평가를 받았다.

### 4.5 학습 흐름 연결성

OX 퀴즈와 오답 복습 기능은 단순한 판례 확인을 넘어서 학습자의 복습 흐름으로 자연스럽게 연결된다는 점에서 학습 앱으로서의 가능성을 보여주었다.

## 5. 한계점

### 5.1 플랫폼 지원 제한

현재 앱은 iOS 기반으로 구현되어 있어 안드로이드 사용자는 이용할 수 없다. 따라서 운영체제 지원 범위가 제한되어 사용자 접근성에 한계가 있다.

### 5.2 초기 사용 안내 부족

앱을 처음 실행했을 때 어떤 기능을 어떤 순서로 사용해야 하는지 바로 이해하기 어렵다는 의견이 있었다. 기능 자체는 직관적이지만, 초기 사용자에게는 간단한 안내나 튜토리얼이 필요하다.

### 5.3 촬영 입력 흐름 부재

현재는 사진에서 이미지를 업로드하는 기능만 제공되며, 카메라로 바로 촬영하여 업로드하는 기능이 없어 실제 사용 흐름에서 불편함이 있다.

### 5.4 화면 표시 문제

판례 요약 결과나 OX 퀴즈 내용이 화면에서 일부 잘려 보이는 문제가 확인되었다. 이는 정보 표시 영역과 스크롤 처리 방식의 개선이 필요한 부분이다.

### 5.5 이동 경로 불편

요약과 퀴즈를 확인한 뒤 홈 화면으로 바로 이동하는 기능이 없어 사용자가 뒤로 가기를 여러 번 눌러야 하는 불편함이 있다.

### 5.6 입력 데이터 부족

입력된 텍스트 양이 적을 경우 AI 요약 결과가 다소 부자연스럽게 나오는 경우가 있었다. 이는 입력 정보가 부족할 때 발생하는 결과로, 최소 입력량 안내나 예외 처리 로직이 필요하다.

### 5.7 검색 기능 범위 한계

앱은 전체 판례를 검색하는 대규모 판례DB 서비스가 아니라, 사용자가 스캔하거나 입력한 판례 자료를 개인 학습 데이터로 저장하고, 이후 복습과 재검색에 활용하는 구조이다. 따라서 검색 기능은 전체 판례 검색보다는 개인 학습 이력 기반 복습 기능에 가깝다.

## 6. 향후 개선 방향

### 6.1 Android 지원 확대

안드로이드 버전 개발을 통해 더 많은 사용자가 앱을 이용할 수 있도록 지원 범위를 확대할 필요가 있다.

### 6.2 온보딩 및 초기 안내

앱 첫 실행 시 간단한 온보딩 화면이나 가이드 팝업을 제공하여 사용자가 OCR, 요약, OX 퀴즈, 오답 복습 기능을 어떤 순서로 사용하면 되는지 쉽게 이해할 수 있도록 개선할 계획이다.

### 6.3 촬영 입력 기능 추가

사진 업로드뿐 아니라 카메라로 바로 촬영해 판례 자료를 입력할 수 있는 기능을 추가하여 실제 학습 상황에서의 사용 편의성을 높일 수 있다.

### 6.4 화면 표시 안정성 강화

판례 요약과 OX 퀴즈 결과가 잘리지 않도록 화면 레이아웃과 스크롤 영역을 개선하고, 긴 텍스트도 안정적으로 확인할 수 있도록 UI를 보완할 필요가 있다.

### 6.5 빠른 이동 경로 제공

요약과 퀴즈를 모두 확인한 뒤 홈 화면으로 바로 이동할 수 있는 버튼을 추가하여 사용자의 이동 경로를 단축할 수 있다.

### 6.6 입력량 예외 처리 강화

입력 텍스트가 부족한 경우에는 “요약을 위해 더 많은 판례 내용이 필요합니다”와 같은 안내 문구를 제공하거나, 부족한 정보에 맞는 간단 요약 모드로 전환하는 방식의 예외 처리를 강화할 수 있다.

### 6.7 개인 학습 데이터 활용 확대

사용자가 스캔하거나 입력한 판례 자료가 개인 학습 데이터로 누적되는 구조를 적극 활용할 수 있다. 예를 들어 이전에 학습한 판례를 다시 검색하거나, 자주 등장한 키워드와 오답 이력을 바탕으로 개인 맞춤형 복습 목록을 제공하는 방향으로 발전시킬 수 있다.

## 7. 결론

STACK112는 디자인 완성도와 기능 흐름 면에서 긍정적인 평가를 받았다. 향후 안드로이드 지원, 초기 온보딩, 촬영 입력 기능, 화면 표시 안정성, 예외 처리 개선이 이루어진다면 학습 앱으로서 완성도가 더욱 높아질 것으로 판단된다.
