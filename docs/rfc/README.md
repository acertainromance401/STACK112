# RFC (Request for Comments)

RFC는 미래 변경사항에 대한 팀 합의를 문서화하는 절차입니다.

## 언제 RFC를 작성하나?

- 아키텍처 변경
- 대규모 리팩터링
- 외부 의존성 교체
- 운영 정책/워크플로우 변경

## 프로세스

1. Draft: RFC 초안 작성
2. Discuss: GitHub Discussions에 공유
3. Decide: 합의 후 상태 변경
4. Implement: 관련 Issue/PR로 실행

## 파일명 규칙

- `NNNN-title.md`
- 예시: `0001-ondevice-metrics-policy.md`

## 상태

- Draft
- Reviewing
- Accepted
- Rejected
- Superseded
