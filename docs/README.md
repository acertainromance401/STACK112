# Documentation Hub

이 디렉터리는 팀의 비동기 협업을 위한 핵심 문서를 모아둔 허브입니다.

## 필수 문서 카테고리

1. 결정 사항 및 배경 (ADR)
- 위치: `docs/adr/`
- 목적: 무엇(What)보다 왜(Why)를 기록

2. 아키텍처 설계
- 위치: `docs/architecture/`
- 목적: 시스템 구조, 데이터 흐름, 다이어그램 정리

3. API 명세
- 위치: `docs/api/`
- 목적: 인터페이스 계약, 요청/응답 예시, 오류 코드

4. 트러블슈팅 가이드
- 위치: `docs/troubleshooting/`
- 목적: 장애 대응 절차, 자주 발생하는 문제와 해결책

5. 회의 노트
- 위치: `docs/meeting-notes/`
- 목적: 의사결정 내역, 액션 아이템, 담당자/기한

6. FAQ 및 온보딩
- 위치: `docs/onboarding/`
- 목적: 신규 참여자 빠른 적응, 반복 질문 축소

## 현재 문서와 연결

- 프로젝트 기획/로드맵: `docs/project/`
- 역할별 실행 가이드: `docs/roles/`
- 아키텍처 상세: `code/ARCHITECTURE_AND_DEPLOYMENT.md`
- 운영/배포 상세: `code/PRODUCTION_DEPLOYMENT_GUIDE.md`
- 실행 가이드: `code/Run_Guide_AI_SYS.md`
- iOS 트러블슈팅: `code/ios/README.md`

## 업데이트 원칙

- 기능/정책 변경 시 관련 문서를 같은 PR에 함께 업데이트
- ADR은 변경 불가 사실이 아니라, 변경 이유를 명확히 남기는 문서
- 회의 노트에는 결정, 근거, 액션 아이템을 반드시 포함
